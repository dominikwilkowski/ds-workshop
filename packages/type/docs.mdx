---
title: The type package
description: This is what the type is meant to do and why we have it
---

The `Type` package encapsulates all of our typography styles and keeps it in a single place.

## Body font packs

```jsx live
<Stack>
	<Type look="body12" as="p">
		Type as body12
	</Type>
	<Type look="body12bold" as="p">
		Type as body12bold
	</Type>
	<Type look="body14" as="p">
		Type as body14
	</Type>
	<Type look="body14bold" as="p">
		Type as body14bold
	</Type>
	<Type look="body16" as="p">
		Type as body16
	</Type>
	<Type look="body16bold" as="p">
		Type as body16bold
	</Type>
	<Type look="body18" as="p">
		Type as body18
	</Type>
	<Type look="body18bold" as="p">
		Type as body18bold
	</Type>
	<Type look="body20" as="p">
		Type as body20
	</Type>
	<Type look="body20bold" as="p">
		Type as body20bold
	</Type>
	<Type look="body24" as="p">
		Type as body24
	</Type>
	<Type look="body24bold" as="p">
		Type as body24bold
	</Type>
</Stack>
```

## Chose the tag to render

```jsx live
<Stack>
	<Type look="heading30">Render as span</Type>
	<Type look="heading30" as="p">
		Render as paragraph
	</Type>
	<Type look="heading30" as="div">
		Render as div
	</Type>
</Stack>
```
